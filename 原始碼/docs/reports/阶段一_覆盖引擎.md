# 阶段一汇报 · 测试用例覆盖最大化引擎

## 背景与目标
此前测试用例的**数量和覆盖范围完全依赖大模型自觉**:同一章节,模型给几条、覆盖哪些边界,每次都不一样,既不稳定也无法证明"测全了"。本阶段引入一层**确定性枚举引擎**,把"测哪些点"从模型手里收回到程序,模型只负责把每个点填写成规范卡片。

## 做了什么
1. **边界值分析(BVA)**:对每个带上下限的参数,自动生成 7 个测试点——下界-误差、下界、下界+误差、正常值、上界-误差、上界、上界+误差。误差按参数类型走配置(电压静态 ±10mV / 电压充放电 ±30mV / 电流 ±10mA / 温度 ±1°C)。
2. **成对组合(pairwise)**:多个参数时,用成对覆盖算法(allpairspy)覆盖参数间的**所有两两组合**,而不必全量穷举——例如 3 个各 7 取值的参数,全穷举 343 条,pairwise 约 50 条即可覆盖全部两两交互。
3. **约束过滤**:自动排除非法/互斥组合(如 min>max、空值)。
4. **全程离线**:组合计算一律本地完成,绝不调用任何在线接口。

## 技术实现
- 新增 `coverage.py`:边界点生成、参数类型识别、pairwise 组合、约束过滤。
- 新增接口 `POST /api/cases/{id}/coverage`:直接返回程序枚举出的测试点计划(数量可量化、可复现),**不调用模型即可证明覆盖**。
- 生成支持 `mode=engine`:程序定测试点 → 模型逐条填写;测试控制台加了"覆盖引擎"开关。

## 效果与价值
- 测试点的**数量与覆盖由程序保证、可量化、可复现**,不再随模型波动。
- 用 pairwise 在"覆盖充分"和"用例不爆炸"之间取得平衡。
- 测试质量从"看模型心情"升级为"工程可控"。

## 演示步骤
1. 打开测试控制台,上传规格书 PDF + 参数 CSV;
2. 调 `POST /api/cases/{id}/coverage` 查看程序枚举出的测试点清单与数量;
3. 勾选"覆盖引擎"开关再生成,卡片与枚举的测试点一一对应。

## 验收
给定含多参数的章节,引擎产出:每参数边界点齐全、跨参数成对覆盖、非法组合被排除、离线运行。自测通过。
