# 阶段三汇报 · 语义检索与生产部署

## 背景与目标
历史用例越攒越多,要能"找回来";平台也要能稳妥地在内网上线。本阶段做**检索**与**部署交付**。

## 做了什么
1. **关键词检索**:对生成结果内容与规格书文件名做关键词匹配,返回命中片段。
2. **语义检索(向量)**:用本地嵌入模型(BGE-M3)把结果转成向量,存入向量数据库(pgvector),按**余弦相似度**检索——用相近描述也能召回相关历史用例。
3. **自动降级**:嵌入模型不可用时,语义检索自动降级为关键词检索,不影响使用。
4. **生产部署方案**:nginx 统一入口反向代理,SSE 路径关闭缓冲保证实时,设置上传体积上限,严禁裸暴露后端端口;部署步骤文档化。

## 技术实现
- 新增 `embeddings.py`:OpenAI 兼容嵌入调用,失败返回空以触发降级。
- 新增接口:`GET /api/search?q=&mode=auto|keyword|semantic`;`POST /api/admin/reindex` 为历史结果灌入向量。
- 语义检索基于 pgvector 余弦距离;测试控制台加检索框。
- 交付物:`deploy/nginx.conf`(反代 + SSE 关缓冲 + 上传上限 + 不裸暴露)、`docs/DEPLOYMENT.md`(从装库到上线的完整步骤)。

## 效果与价值
- 历史成果可被**关键词 + 语义**两种方式找回,语义检索对"换种说法"也能召回。
- 一套可复用的内网部署方案,统一入口、实时不卡、安全收敛。

## 演示步骤
1.(首次)`POST /api/admin/reindex` 灌入向量;
2. 控制台检索框输入"过温 / OTC / 短路",分别用关键词与语义模式检索,语义模式显示相似度并按相关性排序;
3. 按 `docs/DEPLOYMENT.md` 经 nginx 走通整条链路。

## 验收
关键词与语义检索均能召回;嵌入不可用时自动降级;经 nginx 反代完成全流程、SSE 不被缓冲。自测通过。

---

## 三阶段总验收
浏览器经统一入口完成 上传 → 勾选 → 排队 →(实时进度)生成 → 查看 → 导出 → 检索 全流程;多人并发零串扰;覆盖引擎保证数量与覆盖且离线运行;历史全局可查、可检索、可导出;模型端点可配置切换;部署文档可复用。
